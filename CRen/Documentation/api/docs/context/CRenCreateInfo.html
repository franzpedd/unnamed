CRenCreateInfo

File: cren_context.h

Type: C Struct

Description

Contains all initialization parameters, resource requests, and bootstrapping requirements in a single data structure. 

typedef struct CRenCreateInfo
{
	const char* appName;
	const char* assetsPath;
	CRen_Renderer api;
	uint32_t appVersion;
	uint32_t width;
	uint32_t height;
	CRen_MSAA msaa;
	bool validations;
	bool vsync;
	bool customViewport;
} CRenCreateInfo;

Members

appName:

Your application/game name, it's used internally by the Vulkan initialization..

assetsPath:

Root path to where all your application/game assets are, must also include CRen's own assets inside of it.

api:

Which renderer API is going to be initialized, currently only vulkan version 1.1, 1.2 and 1.3 are available.

appVersion:

Numerical version for the application, you may want to use a macro for that, like:

#define MAKE_VERSION(variant, major, minor, patch) ((((uint32_t)(variant)) << 29U) | (((uint32_t)(major)) << 22U) | (((uint32_t)(minor)) << 12U) | ((uint32_t)(patch)))

width:

Initial renderer width, usually this should match the window initial width.

height:

Initial renderer height, usually this should match the window initial height.

msaa:

Multisample algorithm, could be from CREN_MSAA_X1, CREN_MSAA_X2 up to CREN_MSAA_X64 in power of two.

validations:

Enables/disables GPU logs about any error or information, this will make the application be slower but is excential when developing.

vsync:

Enables/disables vertical syncronization, which will match GPU to monitor refresh rate.

customViewport:

Hints the renderer the're will be a custom viewport, usually used on Engine Editors, that's quite a complex subject but it's used on Cosmos Engine as an example.
