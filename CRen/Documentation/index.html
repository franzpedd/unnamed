Introduction

What is CRen?

CRen is a 3D rendering engine that provides easy-to-use functions for making both 2D/3D applications, abstracting low-level rendering API calls. It's designed to be fast and reliable, open-source and free.

What CRen isn't?

CRen is not a full game engine, only a graphics framework. It currently provides rendering capabilities exclusively, without physics, AI, sound, or networking systems, making it perfect for integration as the visual component in custom engine architectures.

Features

  • Written in C99, compiles in MSVC and Clang.
  • Support for: Windows, Linux and Android.
  • Handles multiple frames simultaneosly.
  • Abstracts Vulkan but provide access if needed.
Note: This project is in early development. Features will change in future releases.

Installation

CRen is a library, follow these steps to set-it up:

Prerequisites

  • CMake 3.22.1 or higher.
  • MSVC or Clang.
  • Vulkan SDK.

Building Steps

  1. Clone the repository: 
    git clone https://github.com/franzpedd/unnamed.git
  2. Navigate to the project's build directory: 
    cd unnamed/Build
  3. Build the project with cmake 
    cmake ..
Note: Build it with CMake flag -DBUILD_PROJECTS=OFF in order to only generate the CRen library.

Usage

1. Include cren's header file:

#include "cren.h"

2. Setup the initialization structure:

CRenCreateInfo createInfo = { 0 };
createInfo.appName = "Your game/engine/application name";
createInfo.assetsPath = "Assets root directory to look for files, must also include CRen's Build/Bin/data files";
createInfo.api = CREN_RENDERER_API_VULKAN_1_1; // can be requested 1_0, 1_1, 1_2 and 1_3
createInfo.width = 1920; // initial renderer width, must match with window's width, can be changed latter
createInfo.height = 1080;// initial renderer height, must match with window's height, can be changed latter
createInfo.msaa = CREN_MSAA_X4; // Multisample algorithm, can be use any power of 2 from X1 to X64
createInfo.validations = true; // turns on/off trace/info/warning/error messages from vulkan side.
createInfo.vsync = false; // turns on/off vertical synchronization, which will match rendering sync with monitor sync
createInfo.customViewport = true; // hints the renderer the game's window will be on a dedicated viewport, like an editor viewport

3. Initialize CRen's context:

CRenContext* context = cren_initialize(ci); // can be checked for null for success, will also prompt console messages if any error occured

4. Setup the callbacks:

// CRen doesn't handle the window, allowing the usage of GLFW/SDL/etc... but the renderer must recieve/send information from/to it and for that we must setup some callbacks
// These are all callbacks available that must be set-up for a proper application, in the future this won't be necessary.
cren_set_user_pointer(context, this); // (This is Optional, for C++ working with lambdas)
cren_set_resize_callback(context, OnResizeCallback);
cren_set_draw_ui_raw_data_callback(context, OnDrawUICallback);
cren_set_ui_image_count_callback(context, OnSetUIImageCountCallback);
cren_set_get_vulkan_instance_required_extensions_callback(context, OnGetVulkanInstanceRequiredExtensionsCallback);
cren_set_create_vulkan_surface_callback(context, OnCreateVulkanSurfaceCallback);
cren_set_render_callback(context, OnRenderCallback);

You can check the Examples on how to set this up.

Frequently Asked Questions

Why Vulkan, and which version?

Vulkan enables CRen to deliver high performance across multiple platforms. By supporting Vulkan 1.1 as the baseline, we ensure compatibility with older systems without sacrificing the API's modern rendering capabilities.

Why does it need callbacks?

CRen is windowing-system agnostic. This gives you the freedom to choose any windowing system, just as you had the freedom to choose CRen itself.

Will it have 'X' feature?

CRen is currently in early development, so many features you might expect from other solutions are still being implemented. However, with sufficient development time and community feedback, we're committed to adding the features that matter most.
